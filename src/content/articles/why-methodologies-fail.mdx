# Why CSS methodologies fail

Most CSS methodologies start with good intentions.

Over time, many become rigid systems disconnected from how the web platform
actually evolves.

## Common failure patterns

- excessive naming verbosity
- architectural dogma
- over-engineering
- fighting the platform
- utility overload
- framework dependence
- deeply coupled components

## The real issue

Many methodologies attempted to compensate for missing browser features.

Modern CSS now includes:

- cascade layers
- scope
- nesting
- container queries
- logical properties
- modern selectors

Architecture should evolve alongside the platform.

## LSCSS direction

LSCSS focuses on:

- semantic readability
- predictable layers
- low specificity
- shallow selectors
- modern platform features
- gradual migration

The goal is not rigid doctrine.

The goal is maintainable systems.
